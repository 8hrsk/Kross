# Kross Client Integration Guide

Integrating Kross into your desktop application involves three main steps:
1. Initializing the library and conducting an offline check during startup.
2. Displaying the Fyne activation window (GUI) if no valid license exists.
3. Starting background server synchronization to query for revocations and update status.

---

## 🛠️ Step 1: Initializing the Client

First, import the `github.com/user/kross/pkg/client` and `github.com/user/kross/gui` packages.

Define your **public key** (contents of `public.key`), which you generated using the CLI:

```go
package main

import (
	"context"
	"crypto/ed25519"
	"fmt"
	"log/slog"
	"os"
	"time"

	"github.com/8hrsk/Kross/pkg/client"
	"github.com/8hrsk/Kross/gui"
)

// Public key generated by the developer. Replace with your actual bytes.
var appPublicKey = []byte{
    // Copy the bytes from your public.key here
}

func main() {
	logger := slog.New(slog.NewJSONHandler(os.Stdout, nil))

	// 1. Initialize the Kross client
	c, err := client.New(client.Config{
		PublicKey: appPublicKey,
		ServerURL: "http://licensing.my-app.com:8080", // Your server URL
		AppName:   "MyAwesomeApp",                     // Folder name for cache
	})
	if err != nil {
		logger.Error("Failed to initialize licensing client", "err", err)
		os.Exit(1)
	}

	// 2. Perform offline verification
	if err := c.CheckLicense(); err != nil {
		logger.Warn("License not found or invalid. Launching GUI activation...", "reason", err)

		// 3. Open the GUI window to request Email and Key
		// This blocks execution until the window is closed or activation succeeds
		success := gui.ShowActivationWindow(c, "MyAwesomeApp")
		if !success {
			logger.Error("Activation cancelled by the user. Exiting.")
			os.Exit(1)
		}
	}

	// 4. Start background syncing
	ctx, cancel := context.WithCancel(context.Background())
	defer cancel()
	
	// Periodically sends local activations to the server and updates the blacklist
	c.StartBackgroundSync(ctx, logger)

	logger.Info("License successfully validated! Starting app...")
	
	// Launch your main application logic
	runAppLogic()
}

func runAppLogic() {
	// ... Your app code here ...
	time.Sleep(10 * time.Second)
}
```

## 🔒 Client-Side Storage Security & Encryption

The license cache is saved locally on disk and fully encrypted.
- **Encryption**: Stored license data (`license.dat`), the background sync queue, and local blacklists are encrypted using **AES-256-GCM**.
- **Key Derivation**: The key is generated via `SHA256(HWID + AppName)`. This links the cache file directly to the machine's hardware, meaning users cannot copy their activated license file to a different computer.
- **Default Storage Directories**:
  - **Windows**: `%APPDATA%\Kross` folder + a mirrored backup record in the Windows Registry (`HKCU\Software\Kross`) to safeguard against accidental file deletions.
  - **macOS**: `~/Library/Application Support/.kross`
  - **Linux**: `~/.config/.kross`

## 🛡️ Obfuscation & Decompilation Prevention

To prevent hackers from bypassing license checks (such as modifying assembly to jump over the `CheckLicense` call), build your Go project using the **Garble** tool rather than standard `go build`.

`Garble` applies extensive obfuscation:
- Encrypts literal strings (preventing extraction of server URLs or message logs).
- Scrambles function, type, variable, and package names into random strings.
- Strips debug symbols, compiler pathways, and type metrics.

### Installing Garble:
```bash
go install mvdan.cc/garble@latest
```

### Compiling Obfuscated Releases:
```bash
garble -literals -tiny -seed=random build -ldflags="-s -w" -trimpath -o dist/my-app ./cmd/my-app/main.go
```

**Compilation Flags Description:**
- `-literals` — Obfuscates all numbers and strings.
- `-tiny` — Strips metadata to reduce the footprint.
- `-seed=random` — Generates a unique codebase structure for every build.
- `-ldflags="-s -w"` — Strips debugging tables (Symbol Table) and DWARF descriptors.
- `-trimpath` — Clears host computer directory references from compilation stacks.
