# Интеграция Kross в Ваше приложение

Интеграция Kross состоит из трех основных шагов:
1. Инициализация и офлайн-проверка лицензии при старте.
2. Подключение графического окна активации (GUI на Fyne) в случае отсутствия или невалидности лицензии.
3. Настройка фоновой синхронизации с сервером для отслеживания блокировок.

---

## 🛠️ Шаг 1: Подключение клиентской библиотеки и запуск

Сначала импортируйте пакеты `github.com/user/kross/pkg/client` и `github.com/user/kross/gui` в ваше приложение.

Вставьте **публичный ключ** (содержимое файла `public.key`), сгенерированный с помощью CLI:

```go
package main

import (
	"context"
	"crypto/ed25519"
	"fmt"
	"log/slog"
	"os"
	"time"

	"github.com/8hrsk/Kross/pkg/client"
	"github.com/8hrsk/Kross/gui"
)

// Публичный ключ, экспортированный разработчиком. Замените на ваши байты.
var appPublicKey = []byte{
    // Сюда необходимо скопировать байты вашего public.key
}

func main() {
	logger := slog.New(slog.NewJSONHandler(os.Stdout, nil))

	// 1. Инициализируем клиент Kross
	c, err := client.New(client.Config{
		PublicKey: appPublicKey,
		ServerURL: "http://licensing.my-app.com:8080", // URL вашего сервера
		AppName:   "MyAwesomeApp",                     // Название папки с кэшем лицензии
	})
	if err != nil {
		logger.Error("Ошибка инициализации лицензирования", "err", err)
		os.Exit(1)
	}

	// 2. Проверяем наличие локальной валидной лицензии
	if err := c.CheckLicense(); err != nil {
		logger.Warn("Лицензия не найдена или недействительна. Открываем окно активации...", "reason", err)

		// 3. Открываем окно GUI для ввода данных (почта + ключ)
		// Этот метод блокирует поток, пока окно не закроют или лицензия не будет успешно активирована
		success := gui.ShowActivationWindow(c, "MyAwesomeApp")
		if !success {
			logger.Error("Активация отклонена пользователем. Завершение работы.")
			os.Exit(1)
		}
	}

	// 4. Запускаем фоновую синхронизацию с сервером
	ctx, cancel := context.WithCancel(context.Background())
	defer cancel()
	
	// Синхронизирует отложенные активации и проверяет черный список
	c.StartBackgroundSync(ctx, logger)

	logger.Info("Лицензия успешно проверена! Запуск приложения...")
	
	// Основная логика вашей программы
	runAppLogic()
}

func runAppLogic() {
	// ... Ваше приложение работает здесь ...
	time.Sleep(10 * time.Second)
}
```

## 🔒 Безопасность и шифрование данных на клиенте

Кэш лицензии хранится локально на диске в зашифрованном виде.
- **Шифрование**: Данные лицензии (`license.dat`), а также очереди синхронизации и черного списка шифруются алгоритмом **AES-256-GCM**.
- **Ключ шифрования**: Вычисляется как хэш `SHA256(HWID + AppName)`, что делает невозможным перенос зашифрованного файла лицензии на другое устройство.
- **Пути хранения по умолчанию**:
  - **Windows**: Папка `%APPDATA%\Kross` + дублирование зашифрованной записи в реестр Windows (`HKCU\Software\Kross`) для предотвращения удаления файлов.
  - **macOS**: `~/Library/Application Support/.kross`
  - **Linux**: `~/.config/.kross`

## 🛡️ Защита от декомпиляции и обфускация

Для защиты вашего Go-приложения от взлома и реверс-инжиниринга (например, вырезания вызова `CheckLicense` через ассемблер) используйте инструмент **Garble** вместо стандартного компилятора `go build`.

`Garble` производит глубокую обфускацию:
- Шифрует строковые литералы (чтобы взломщик не нашел строки вроде `"License verified!"` или URL сервера).
- Заменяет имена функций, пакетов, типов и переменных на случайные хэши.
- Удаляет отладочную информацию (скрывает стек вызовов и типы данных).

### Установка Garble:
```bash
go install mvdan.cc/garble@latest
```

### Команда для сборки обфусцированного релиза:
```bash
garble -literals -tiny -seed=random build -ldflags="-s -w" -trimpath -o dist/my-app ./cmd/my-app/main.go
```

**Описание флагов сборки:**
- `-literals` — шифрует все строки и числа в коде.
- `-tiny` — уменьшает размер бинарного файла за счет удаления избыточной метаинформации.
- `-seed=random` — делает каждую новую сборку уникальной по структуре кода.
- `-ldflags="-s -w"` — вырезает отладочные символы (Symbol Table) и DWARF-секции.
- `-trimpath` — удаляет пути к файлам исходного кода с вашего ПК из бинарника.
