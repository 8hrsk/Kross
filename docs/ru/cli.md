# Руководство по Kross CLI

Инструмент командной строки Kross CLI (`cmd/kross-cli`) используется разработчиками приложения для управления криптографическими ключами и выпуска лицензий клиентам.

## Сборка

Вы можете собрать CLI с помощью следующей команды из корня проекта:
```bash
go build -o dist/kross-cli ./cmd/kross-cli
```

---

## Команды и использование

CLI инструмент поддерживает две основные подкоманды: `keygen` и `issue`.

### 1. Генерация ключей: `keygen`
Генерирует новую пару ключей Ed25519 для подписи лицензий. 

```bash
kross-cli keygen
```

**Результат выполнения:**
- В текущей рабочей директории создаются два файла:
  - `private.key` — закрытый ключ (сохраняйте в секрете, используется для генерации лицензий).
  - `public.key` — открытый ключ (встраивается в код клиента для валидации лицензий).
- Файлы записываются в PEM-формате.

---

### 2. Выпуск лицензии: `issue`
Используется для выпуска лицензионных ключей. Команда считывает `private.key` для подписи структуры лицензии и выводит закодированный ключ.

#### Параметры:
- `--key` (необязательный) — путь к секретному ключу (по умолчанию `./private.key`).
- `--email` (необязательный) — электронная почта клиента для персональной лицензии.
- `--days` (необязательный) — срок действия лицензии в днях. Если не указан или `<=` 0, лицензия считается бессрочной.
- `--mass` (необязательный) — количество генерируемых массовых (одноразовых) ключей.

#### Вариант А: Персональная лицензия (привязана к Email)
Используется для привязки к конкретному пользователю. Клиентское приложение будет сверять введенный пользователем email с зашитым в ключе.

```bash
kross-cli issue --email="client@example.com" --days=365
```
**Вывод в stdout:**
```text
KROSS-MY5TA-PBP2E-FZX4A-JD7Q4-S46NX-V3DZO-U6GQM-D2H2I-C3SDA-Q===
```

#### Вариант Б: Бессрочная персональная лицензия
```bash
kross-cli issue --email="vip@example.com"
```

#### Вариант В: Массовый выпуск ключей (Mass-activation keys)
Используется для дистрибуции ключей без предварительной привязки к email (например, продажа через сторонние магазины). Ключи являются одноразовыми: при первой активации они связываются с email и HWID на сервере, повторная активация на другом ПК заблокирует этот ключ.

```bash
kross-cli issue --mass=150
```
**Результат выполнения:**
- Создается текстовый файл `licenses_YYYYMMDD_HHMMSS.txt` в текущей директории.
- В файл записывается 150 уникальных ключей (по одному на строку).
- Выводится путь к созданному файлу и количество сгенерированных ключей.
